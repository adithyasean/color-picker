## üì± What is this App?

**Color Picker** (formerly Chroma Match) is a memory matching game built for iPhone. 

**The Goal:**
The player is presented with a grid of face-down cards. They must flip two cards at a time to find matching colors.
- **Match:** The cards stay face up, and you get points (+2).
- **Mismatch:** The cards flip back down, and you lose a point (-1).
- **Win:** Find all pairs to finish the game and save your high score!

## üìÇ Project Structure (Where things live)

If you open the project folder, here is what each file actually does in plain English:

### 1. The "Brains" & Logic
*   **`Card.swift`**: 
    *   This defines what a "Card" is. It knows its Color, whether it is Face Up, and if it is Matched.
*   **`ScoreManager.swift`**: 
    *   This is the scoreboard. It saves your High Scores to the phone's permanent storage (so they survive even if you close the app).

### 2. The User Interface (Screens)
*   **`ColorPickerApp.swift`**: 
    *   The "Main Entrance". This is the first file the app runs. It sets up the main window.
*   **`HomeView.swift`**: 
    *   The **Start Screen**. It shows the animated gradient background, the Title, and the "Play Game" / "High Scores" buttons.
*   **`GameView.swift`**: 
    *   The **Actual Game**. This file contains the logic for the grid of cards, flipping animations, scoring math, and haptic feedback. It's the biggest file because it handles the gameplay.
*   **`CardView.swift`**: 
    *   A small helper that draws **one single card**. It handles the 3D flip animation and drawing the question mark on the back or the color on the front.
*   **`HighScoresView.swift`**: 
    *   The **Leaderboard Screen**. It reads the saved scores from `ScoreManager` and displays them in a pretty list.

### 3. Assets (Images & Colors)
*   **`Assets.xcassets`**: 
    *   A folder containing the **App Icon**, custom colors, and any images used in the app.

## üìù Glossary for Non-Coders

*   **Variable (`var`)**: A box where we store information that can change (like the current `score`).
*   **Function (`func`)**: A set of instructions packaged together. For example, `restartGame()` is a function that sets the score to 0 and shuffles the cards.
*   **View**: Anything visible on screen (a button, a text label, an image).
*   **State (`@State`)**: Special storage for UI. When a `@State` variable changes (like `score` goes from 0 to 2), the screen *automatically* redraws itself to show the new number. You don't have to tell it to update!
